Troubleshooting

Common issues and how to resolve them.

FixAI model not responding

If using a cloud model, verify your API key is correct and has not expired. Check your internet connection. If using a local model, ensure the model file was fully downloaded and is compatible. Try switching to the default local LLM to confirm the app is working.

FixImage analysis not working

Make sure you have granted camera and storage permissions. If your AI model doesn't support image recognition, the app will automatically use ML toolkit for text extraction — ensure the image contains readable text. For best results with images, use a cloud model with vision support (e.g., GPT-4o, Gemini).

FixGmail connection failed

Ensure you are using an app password, not your regular Gmail password. You can generate an app password at myaccount.google.com/apppasswords. Two-factor authentication must be enabled on your Google account to create an app password.

FixInternet search returns no results

Check your internet connection. If you have a filter selected (News, Location, Images, or Video), try deselecting it to perform a general web search instead. Some queries may return limited results depending on the search category.

TipCustom API not connecting

When adding a custom OpenAI-compatible API, double-check the endpoint URL includes the full path (e.g., http://localhost:11434/v1 for Ollama). If using a local server, make sure it's running. For bearer token authentication, verify the API key is correct. Use the quick presets (Ollama, LM Studio, OpenRouter) for auto-filled configurations.

TipTranscription quality is low

Speak clearly and minimize background noise. The transcription engine works best in quiet environments. If possible, hold your device closer to the speaker. After transcription, use the AI Summarize feature to clean up and structure the output.
